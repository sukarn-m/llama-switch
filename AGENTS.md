Guidance for AI coding agents working in the `llama-switch` codebase.

## What this project is

`llama-switch` is a **GPU model multiplexer for `llama-server`** (from llama.cpp). It is an HTTP reverse proxy that presents a single OpenAI-compatible endpoint and transparently loads/unloads different GGUF models on one or more GPUs behind the scenes. It was written to replace a shell script (`llama-server.sh`) that ran a single fixed model ‚Äî now many models can coexist and be switched on demand within VRAM and concurrency limits.

Key idea: a client sends `POST /v1/chat/completions` with `{"model": "my-model", ...}`. The proxy parses the `model` field, ensures that model's `llama-server` backend is running (spawning it if needed, evicting least-recently-used models to make room), and forwards the request ‚Äî including SSE streaming ‚Äî to that backend on a dynamically-allocated loopback port.

It is a **single-binary Go program** with no subpackages and one third-party dependency (`gopkg.in/yaml.v3`). Target environment is NVIDIA CUDA multi-GPU Linux (it shells out to `nvidia-smi` for VRAM accounting).

## Repository layout

Everything lives at the repo root in `package main`. There are no subdirectories and no internal packages.

## Architecture & request flow

```
Client ‚îÄ‚îÄHTTP‚îÄ‚îÄ‚ñ∂ ProxyServer (:8080)
                   ‚îÇ
                   ‚îú‚îÄ GET  /                ‚Üí health
                   ‚îú‚îÄ GET  /v1/models       ‚Üí OpenAI model list (with status field)
                   ‚îú‚îÄ GET  /models          ‚Üí llama.cpp router model list (with status)
                   ‚îú‚îÄ POST /models/load     ‚Üí explicitly load a model {"model":"id"}
                   ‚îú‚îÄ POST /models/unload   ‚Üí explicitly unload {"model":"id"} (Open WebUI "Eject")
                   ‚îú‚îÄ GET  /v1/loaded       ‚Üí legacy: list loaded models only
                   ‚îî‚îÄ everything else       ‚Üí handleProxy
                                            ‚îÇ
                    parse "model" from JSON body
                                            ‚ñº
                   BackendManager.EnsureLoaded(modelID)
                     ‚îú‚îÄ already loaded? ‚Üí reuse
                     ‚îú‚îÄ else: ensureCapacityLocked
                     ‚îÇ    ‚îú‚îÄ enforce MaxModels (evict LRU)
                     ‚îÇ    ‚îî‚îÄ enforce VRAM + 2GB headroom (evict LRU)
                     ‚îú‚îÄ startBackendLocked: alloc port, spawn llama-server,
                     ‚îÇ    wait for /health 200 (HealthTimeoutSeconds)
                     ‚îî‚îÄ return *Backend
                                            ‚ñº
                   forward() ‚Üí proxy to 127.0.0.1:<backend port>,
                   stream response back if "stream":true
```

- Each loaded model = one `llama-server` child process on its own port from `backend_port_start`‚Äì`backend_port_end`.
- Backends are addressed only on loopback (`127.0.0.1`); the public listener binds to the configured `host` (set in `config.yaml`).
- The idle sweeper goroutine (`idleSweeper`) unloads any backend untouched for `idle_timeout_minutes`, polled every `sweep_interval_seconds`.

## Configuration reference (`config.yaml`)

- `server`: `host`, `port`, `backend_port_start/end`, `max_models`, `idle_timeout_minutes`, `health_timeout_seconds`, `sweep_interval_seconds`.
- `backend`: `binary` (path to `llama-server`), `env` (notably `LD_LIBRARY_PATH` for CUDA libs), `common_args` (flags prepended to every model invocation), `workdir`, `nvidia_smi`.
- `models`: list of `ModelConfig`. Lookup by `id`, `alias`, or `model` name (`FindModel` matches all three). Each builds its llama-server args via `ModelConfig.BuildArgs`. Supports `mmproj` (multimodal), `tensor_split`, `ctx_checkpoints`, and freeform `extra_args`.
- Path expansion: `~` and `$VAR` are expanded in config values (`expand()`), including colon-separated `LD_LIBRARY_PATH`-style lists.

**Defaults applied when fields are zero/empty:** `backend_port_start=8201`, `backend_port_end=8299`, `max_models=2`, `idle_timeout_minutes=60`, `health_timeout_seconds=180`, `sweep_interval_seconds=15`, `nvidia_smi="nvidia-smi"`.

## Commands

```
llama-switch serve [config]    # start proxy (default if no command given)
llama-switch profile [config]  # load each model once, measure VRAM, save to vram-cache.json
llama-switch models [config]   # list configured models + VRAM estimates
llama-switch status [config]   # query a running server's /v1/loaded  (note: wired in usage but dispatched via `serve` path; see main.go)
llama-switch help
```

Config path resolution: positional arg ‚Üí `$LLAMA_SWITCH_CONFIG` ‚Üí `./config.yaml`.

## Conventions

- **Single package, flat layout.** All code is `package main` in the repo root. Do not introduce subpackages without reason; the project's style is one file per concern.
- **Go 1.26** toolchain; only stdlib + `yaml.v3`. Avoid adding dependencies for things the stdlib can do.
- **Comment style:** section dividers use `// ‚îÄ‚îÄ Name ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ` banners; exported functions carry doc comments; `Caller must hold <mu>` notes annotate lock contracts ‚Äî respect these when editing.
- **Locking:** `BackendManager.mu` guards the `backends` map and port allocator interactions during lifecycle. Methods suffixed `Locked` require the caller to hold the lock (e.g. `startBackendLocked`, `stopLocked`, `ensureCapacityLocked`, `evictLRULocked`). Preserve the double-checked-locking pattern in `EnsureLoaded` (RLock fast path ‚Üí Lock slow path).
- **Process output** from each backend is piped through `logPipe` and printed to stdout with a `[model-id]` prefix ‚Äî keep this; it's how service logs are read.
- **Streaming detection** in `forward()` is a naive string scan for `"stream":true` / `"stream": true`. If you touch request handling, be aware this is intentionally simple and does not fully parse JSON.
- **Errors during backend start** (port alloc fail, binary missing, health timeout) must release the port and clean up ‚Äî mirror the cleanup in `startBackendLocked`.
- **Config is authoritative.** `config.yaml` reflects a real deployment with specific GGUF paths, GPU topology, and a bind address. Treat it as live configuration, not an example template.
- **No tests currently exist.** If adding code, consider that there is no existing test harness to extend ‚Äî `go test ./...` will pass trivially today.

## Common tasks for agents

- **Add a new model:** add an entry under `models:` in `config.yaml`; run `llama-switch profile` so admission control has VRAM estimates.
- **Add a new HTTP endpoint:** add a path branch in `ProxyServer.handle` (`proxy.go`) and a `handleXxx` method. Management/model-list endpoints return JSON; proxying goes through `handleProxy` ‚Üí `forward`.
- **Change eviction policy:** `ensureCapacityLocked` and `evictLRULocked` in `backend.go`. LRU is based on `Backend.lastReq`, updated via `Touch()` on every proxied request.
- **Change CLI surface:** `main.go` `switch cmd` block + `usageText`. `parseConfigArg` handles config-path discovery.
- **Rebuild after edits:** `go build` and replace the `llama-switch` binary; restart the systemd unit if running as a service.

## Things to watch out for

- `ReadTimeout`/`WriteTimeout` on the proxy server are intentionally `0` to allow long-lived streaming requests ‚Äî do not set finite timeouts without accounting for streaming.
- The 2 GB VRAM headroom constant (`2048` in `ensureCapacityLocked`) is hardcoded; adjust there if changing the safety margin.
- `waitHealth` polls every 2 s; large models may take close to `health_timeout_seconds` (default 180 s) to become ready. The proxy returns `503` with `Retry-After: 30` if a model fails to load.
- `nvidia-smi` is required for VRAM-aware scheduling; if it is unavailable, `ensureCapacityLocked` falls through optimistically (loads without VRAM checks) ‚Äî keep this fallback intact.
- `config.yaml` contains absolute paths specific to the deployment machine; don't generalize them.
